## 2024-05-18 - Fixing N+1 Query on Kanban Dashboard
**Learning:** Found an N+1 query issue in the `get_kanban_board` endpoint where the backend queried an Application model up to 50 times, but the subsequent code iterating over the results accessed `.job.title` and `.candidate.full_name`. This causes `SQLAlchemy` to load these relationships dynamically, generating an extra query for every single item!
**Action:** Used `joinedload` to eagerly load `.job` and `.candidate` relationships during the initial query to prevent lazily fetching these related models. Remember that if the base query is already filtering with `.join()`, `joinedload` creates an extra redundant JOIN to fulfill its load, so consider using `contains_eager` instead if that JOIN's column aliases and entity selections are handled correctly in the original query!
