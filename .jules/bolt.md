## 2024-03-25 - [Optimize get_cost_centers Database Query]
**Learning:** The application was experiencing severe performance bottlenecks in endpoint `get_cost_centers` because it fetched all raw transactions into memory and instantiated SQLAlchemy models simply to group and sum values by category. Attempting to run a global test suite inside restricted environments required injecting multiple dummy API keys (e.g., OPENAI_API_KEY, GEMINI_API_KEY) and overriding database URLs to in-memory SQLite to prevent failing imports when mocked logic relies on uninitialized external dependencies.
**Action:** Used `func.sum()` and `func.count()` with a `group_by()` clause wrapped in `func.coalesce` at the SQLAlchemy level. When executing global integration tests in isolated environments for FastAPI systems relying on pydantic-settings, always export a comprehensive set of dummy values (`OPENAI_API_KEY`, `DATABASE_URL`) before executing pytest to isolate the test suite reliably.
