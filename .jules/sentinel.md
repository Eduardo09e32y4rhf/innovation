## 2024-05-19 - Ensure Error Messages Do Not Leak Internal Exceptions
**Vulnerability:** Several FastAPI endpoints passed raw Python exceptions (`str(e)`) directly to the `HTTPException` detail fields (e.g., `raise HTTPException(500, detail=str(e))`). This directly exposes stack traces and internal system details (like external service names or failure modes) to external clients.
**Learning:** Returning `str(e)` in error handling blocks is a common anti-pattern that can inadvertently expose sensitive deployment, configuration, or environment details to attackers, violating the principle of failing securely.
**Prevention:** Always log the actual exception details internally using `logger.error` or `print`, and return a generic, sanitized error message to the client in the HTTP response (e.g., `raise HTTPException(500, detail="Erro interno do sistema")`).
