# Repository Structure

Estrutura oficial do MVP:

- `apps/api`: API NestJS, Prisma, autenticacao, funcionarios, ponto, ferias e WhatsApp.
- `apps/web`: frontend Next.js do produto.
- `docs`: documentacao, planejamento e evidencias.
- `infra`: infraestrutura local e arquivos de compose.
- `scripts`: automacoes de suporte ao desenvolvimento.
- `storage`: runtime local ignorado pelo Git; somente placeholders e README devem ser versionados.

Pastas fora desse fluxo sao consideradas legado ou experimentos e nao devem ser usadas por novas implementacoes do MVP.
